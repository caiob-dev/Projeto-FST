Sistema de Controle de Despesas Pessoais (HTML5 / CSS / JavaScript) Nível: Iniciantes (intermediário leve) Tempo estimado: 10–20 horas (dividido em sessões práticas)

Visão geral 
Os alunos irão criar um Sistema de Controle de Despesas Pessoais, utilizando apenas HTML5, CSS e JavaScript puro. O sistema permitirá registrar, listar, filtrar e excluir despesas, além de calcular o total gasto e mostrar um gráfico simples com base nas categorias das despesas.
O projeto é ideal para praticar manipulação do DOM, eventos, armazenamento local e organização de código.

Objetivos de aprendizagem:
● Usar tags semânticas do HTML5. ● Criar uma interface responsiva com CSS (Flexbox e Grid). 
● Manipular o DOM (criar, remover e atualizar elementos dinamicamente). ● Armazenar e recuperar dados do localStorage. 
● Usar funções, objetos e arrays em JavaScript. ● Validar formulários e lidar com eventos. ● Exibir informações de forma visual (gráfico com ).

Pré-requisitos ● Conhecimento básico de HTML, CSS e JavaScript. ● Saber criar e linkar arquivos (index.html, style.css, main.js).

Descrição do projeto O sistema terá as seguintes seções principais:

Header: Título e breve descrição.

Formulário de nova despesa: Campos para descrição, categoria, valor e data.

Tabela de despesas: Lista dinâmica mostrando todas as despesas registradas.

Filtros: Por categoria e por período de tempo.

Resumo: Total de despesas e gráfico simples por categoria.

Footer: Créditos e informações finais.

Funcionalidades obrigatórias ● Adicionar nova despesa (com validação). 
● Listar despesas dinamicamente. ● Excluir despesa. 
● Filtrar despesas por categoria e data. ● Calcular total geral das despesas. 
● Mostrar gráfico simples de pizza (Canvas API) com total por categoria. 
● Salvar e carregar dados via localStorage.

Estrutura de arquivos /despesas-pessoais /css style.css /js main.js /img index.html README.md

Boas práticas: 
● Use funções bem nomeadas e comentários claros. 
● Organize o código em blocos lógicos. 
● Teste no desktop e no mobile. 
● Use toFixed(2) para valores monetários.

Critérios de avaliação 
● Funcionalidade (40%): adicionar, listar, excluir, filtrar, gráfico. 
● Organização do código (20%): clareza, comentários, estrutura. 
● Design e responsividade (20%). ● Usabilidade (10%). 
● Criatividade e melhorias extras (10%).

Extensões opcionais (desafios bônus) 
● Editar despesa existente. ● Adicionar conversão de moeda. ● Salvar preferências no localStorage (tema claro/escuro). 
● Exportar os dados em CSV.

Observação final Este projeto é excelente para consolidar os fundamentos de front-end. O aluno aprende a trabalhar com eventos, lógica de programação, DOM e armazenamento local — pilares essenciais para qualquer desenvolvedor web.
